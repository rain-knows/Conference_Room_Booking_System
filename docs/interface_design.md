# 会议室预约管理系统界面设计文档

## 1. 文档概述

本文档描述了基于现有数据库结构的会议室预约管理系统的界面设计。界面设计遵循直观、易用的原则，确保各类用户能够高效地完成日常操作。

## 2. 用户界面架构

系统界面架构基于用户角色和主要功能模块进行设计，包括以下几个主要部分：

### 2.1 基础架构
- **登录界面**：所有用户的统一入口
- **主控制台**：根据用户角色展示不同的功能模块和操作权限
- **个人中心**：用户个人信息管理和设置

### 2.2 主要功能模块界面
1. **用户管理模块**
2. **会议室管理模块**
3. **预约管理模块**
4. **设备维护模块**
5. **消息与通知模块**
6. **系统日志模块**

### 2.3 基于权限的界面访问控制
系统基于用户类型和权限矩阵，动态调整界面展示内容和操作权限，确保用户只能访问和操作其被授权的会议室类型和功能模块。

## 3. 界面设计详情

### 3.1 登录界面

![登录界面示意图](../assets/images/login_ui.png)

**界面元素**：
- 系统标题
- 用户名输入框
- 密码输入框
- 登录按钮
- 记住密码选项
- 忘记密码链接

**数据关联**：
- 用户表（User）：验证用户名和密码

**功能描述**：
- 用户输入凭据后，系统验证用户身份
- 验证成功，根据用户角色跳转到相应主控制台
- 验证失败，显示错误提示

### 3.2 主控制台

**界面元素**：
- 顶部导航栏（显示用户信息、通知、设置等）
- 侧边菜单（根据用户角色显示可用功能模块）
- 内容区域（显示当前选择的功能模块内容）
- 状态栏（显示系统状态、当前时间等信息）

**数据关联**：
- 用户表（User）：显示用户信息
- 角色表（Role）、权限表（Permission）：确定可访问的功能模块
- 通知表（Notification）：显示未读通知

### 3.3 用户管理模块

#### 3.3.1 用户列表界面
**界面元素**：
- 用户搜索框
- 用户列表表格（显示用户ID、用户名、全名、邮箱、电话、状态等）
- 添加、编辑、删除用户按钮
- 角色分配按钮

**数据关联**：
- 用户表（User）：提供用户基本信息
- 用户角色表（UserRole）：显示用户已分配角色

#### 3.3.2 角色管理界面
**界面元素**：
- 角色列表（显示角色ID、名称、描述）
- 添加、编辑、删除角色按钮
- 权限分配界面（树形结构显示可用权限）

**数据关联**：
- 角色表（Role）：提供角色基本信息
- 权限表（Permission）：显示可分配权限

### 3.4 会议室管理模块

**数据关联**：

#### 3.9.1 会议室类型管理界面

- 会议室类型列表（显示类型代码、类型名称、描述）

**数据关联**：
- 会议室类型表（RoomType）：提供类型基本信息

**会议室类型表格**：

| 类型代码 | 类型名称 | 描述 |
|---------|---------|------|
| NORMAL | 普通会议室 | 基础设施的会议室，适合一般会议使用 |

| VIP | VIP会议室 | 为高管或特殊会议准备的高档会议室 |
| TRAINING | 培训室 | 专为培训活动设计的会议室 |
| CONFERENCE | 大型会议厅 | 可容纳大量人员的大型会议厅 |

**数据关联**：
- 用户类型表（UserType）：提供用户类型信息
- 会议室类型表（RoomType）：提供会议室类型信息
- 用户类型会议室类型权限表（UserTypeRoomTypePermission）：存储权限映射关系

**会议室访问权限映射表格**：

| 用户类型 | 可访问的会议室类型 |
|---------|------------------|
| 普通员工 | NORMAL, TRAINING |
| 部门经理 | NORMAL, ADVANCED, TRAINING |
| 高级管理层 | NORMAL, ADVANCED, VIP, TRAINING |
| 培训师 | NORMAL, TRAINING |
| 行政人员 | NORMAL, ADVANCED, TRAINING, CONFERENCE |
| 系统管理员 | 所有类型 |

## 4. 界面交互流程

### 4.1 会议室预约流程

```
用户登录 -> 进入主控制台 -> 选择"预约管理" -> 点击"新建预约" -> 
选择会议室 -> 设置时间 -> 输入会议信息 -> 邀请参与者 -> 提交预约 -> 
系统通知参与者 -> 预约成功
```

### 4.2 设备维护流程

```
管理员登录 -> 进入"设备管理" -> 选择有问题设备 -> 点击"创建维护记录" -> 
填写维护信息 -> 提交维护记录 -> 维护完成后更新状态 -> 
系统更新设备可用状态
```

### 4.3 基于用户权限的会议室预约流程

```
用户登录 -> 系统读取用户类型和权限配置 -> 
进入预约管理界面 -> 用户选择日期时间搜索会议室 -> 
系统仅显示该用户有权限预约的会议室类型 ->
用户选择符合条件的会议室 -> 完成预约
```

## 5. 响应式设计

本系统界面采用响应式设计，可适应不同设备屏幕尺寸：

1. **桌面端**：完整功能布局，支持高级数据可视化和管理功能
2. **平板设备**：精简布局，保留核心功能，调整控件大小适应触摸操作
3. **移动端**：极简布局，专注于查看预约、接收通知和简单的预约操作

## 6. 无障碍设计

遵循WCAG 2.1标准，确保界面具有良好的无障碍性：

1. 提供适当的颜色对比度
2. 支持键盘导航
3. 为图像和控件提供替代文本
4. 支持屏幕阅读器

## 7. 界面样式指南

### 7.1 配色方案
- **主色**：#4285F4（蓝色）- 表示可信赖和专业
- **辅助色**：#34A853（绿色）- 表示成功和确认
- **警告色**：#FBBC05（黄色）- 表示警告和注意
- **错误色**：#EA4335（红色）- 表示错误和危险
- **中性色**：#9AA0A6、#BDC1C6、#E8EAED、#F8F9FA - 用于文本、背景和边框

### 7.2 字体
- 标题：16-24px，粗体
- 正文：14px，常规
- 小字体：12px，适用于辅助信息
- 字体族：无衬线字体（如Microsoft YaHei UI, Arial等）

### 7.3 组件样式
- **按钮**：圆角矩形，内边距适中，悬停效果
- **表格**：行间隔背景色，突出显示当前行
- **表单**：垂直布局，标签位于字段上方
- **卡片**：轻微阴影效果，圆角边框

## 8. 原型设计参考

为充分展示系统界面效果，建议使用以下原型工具创建高保真原型：

1. **Figma**：用于创建详细的UI设计和交互原型
2. **Adobe XD**：适用于创建响应式界面设计
3. **Axure RP**：适用于创建高交互性的原型

## 9. 实现技术建议

基于当前项目的Java Swing架构，建议：

1. **使用MVC模式**：将数据模型、界面显示和业务逻辑分离
2. **组件化开发**：创建可复用的自定义组件库
3. **利用第三方库**：如JFormDesigner优化界面设计，MigLayout提供更灵活的布局管理
4. **考虑JavaFX迁移**：长期来看，可考虑迁移到JavaFX以获得更现代的界面效果

## 10. 测试与评估建议

1. **可用性测试**：邀请不同角色的用户测试界面
2. **兼容性测试**：在不同操作系统和显示分辨率下测试
3. **性能测试**：评估复杂界面（如大量数据的表格）的响应速度
4. **无障碍性测试**：使用辅助技术验证无障碍功能

## 11. 结论

本界面设计文档围绕现有数据库结构，提出了一套完整的界面设计方案。设计注重用户体验和功能完整性，同时考虑了实现的可行性。界面设计与数据库结构紧密匹配，确保系统各部分协调工作，为用户提供高效、直观的会议室预约管理体验。
